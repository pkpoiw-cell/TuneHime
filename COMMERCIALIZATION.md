# 商业化上线计划

## 上线目标

第一阶段建议上线为 Windows 直播修音工具，主打：

- 实时修音
- 直播软件可接入
- 简单预设
- 低配置电脑可运行
- 出问题可快速回退到旁路

## 当前状态

当前工程已经具备内测版基础：

- 实时音频输入输出
- 修音参数和人声处理链路
- 内置和自定义预设
- 配置持久化
- 日志
- 打包脚本
- 单元测试

## 正式上线前必须完成

1. 替换高质量 pitch shifter。
   当前算法适合验证，不适合商业音质。优先级最高。

2. 解决虚拟麦克风。
   短期可集成 VB-CABLE 配置说明；正式版建议自研虚拟音频驱动、购买授权方案，或先做 OBS/VST3 插件。

3. 加入音频设备兼容测试。
   覆盖 44.1kHz、48kHz、蓝牙耳机、USB 声卡、ASIO/WASAPI 常见场景。

4. 做安装包。
   推荐 Inno Setup 或 WiX。安装包需要包含运行环境、快捷方式、卸载器和日志路径说明。

5. 代码签名。
   Windows 商业发布建议签名，否则用户会遇到 SmartScreen 警告。

6. 用户协议和隐私政策。
   当前版本不上传音频；正式文案应明确本地处理、日志内容和崩溃信息。

7. 崩溃和诊断反馈。
   至少提供“一键打开日志目录”和错误码；如果要上传日志，必须征得用户同意。

## 建议版本路线

### v0.3 内测版

- 替换 pitch shifter
- 增加延迟显示
- 增加一键打开日志
- 增加设备测试向导
- 完成 20 台机器的人工测试

### v0.5 公测版

- 安装包
- 自动更新
- OBS 接入指南
- 预设市场雏形
- 崩溃日志收集

### v1.0 商业版

- 自有虚拟麦克风或 OBS/VST3 插件
- 代码签名
- 付费授权
- 客服诊断工具
- 完整音质 QA

## 商业风险

- 修音算法授权：使用第三方库前确认商业授权。
- 虚拟音频驱动：驱动签名和兼容性成本较高。
- 直播平台规则：避免宣传“违规变声、冒充他人声音”等用途。
- 音频隐私：如果后续加入云端 AI，必须清楚告知用户音频是否上传。
