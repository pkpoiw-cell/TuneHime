# AI Live Tuner 商业化方案

## 市场分析

### 目标用户
- **主力**: 抖音/B站/快手直播唱歌主播（个人）
- **次要**: 小型直播公会、MCN 机构
- **潜力**: 网课老师、播客主持人、短视频配音

### 你的优势

- 一键直播模式，零门槛
- 价格可以远低于 Auto-Tune

## 推荐方案：订阅制 + 免费增值

### 免费版（引流）
```
功能限制:
- 只有"自然"预设
- 修音强度上限 50%
- 无自动调号识别
- 无自定义预设保存
- 每次启动弹一次品牌水印提示
```

### 专业版 ¥29/月 或 ¥199/年
```
全部功能解锁:
- 所有预设（自然/流行/电音）
- 自动调号识别
- 自定义预设保存
- 修音强度/速度全部可调
- 齿音消除、混响
- 无水印
```

### 为什么选订阅制
1. **现金流稳定** — 月入可预测
2. **降低门槛** — ¥29/月 比一次性 ¥299 更容易决策
3. **持续更新** — 用户付费后你有动力迭代
4. **防破解** — 在线验证比离线注册码更安全

## 实施路线

### 第一步：加许可证验证（1-2 周）

```python
# server.py 中加验证
import hashlib
import json
import urllib.request

LICENSE_SERVER = "https://your-api.com/api/verify"

def verify_license(license_key: str) -> dict:
    """验证许可证，返回功能权限"""
    try:
        resp = urllib.request.post(LICENSE_SERVER, json={
            "key": license_key,
            "hwid": get_machine_id(),
        })
        data = json.loads(resp.read())
        return {
            "valid": data.get("valid", False),
            "plan": data.get("plan", "free"),
            "expires": data.get("expires"),
        }
    except Exception:
        return {"valid": False, "plan": "free"}
```

### 第二步：前端加付费墙（3-5 天）

在 Electron 前端中：
- 免费版：滑块有上限，部分按钮灰色
- 点击付费功能 → 弹出升级提示 → 跳转支付页面
- 支持微信/支付宝扫码支付

### 第三步：后端 API（1 周）

```
POST /api/verify     → 验证许可证
POST /api/purchase   → 创建订单
GET  /api/user       → 查询用户信息
```

用 Supabase 或 Firebase 做后端，省去自建服务器。

### 第四步：支付接入（1 周）

国内首选：
- **微信支付** — 覆盖率最高
- **支付宝** — 补充
- **Stripe** — 如果做海外

小程序/H5 支付最简单，不需要 App 资质。

## 其他收费方式（可选）

### 方案 B：一次性买断 ¥199-299
- 适合不想维护订阅系统的团队
- 缺点：收入不稳定，用户买完就不付费了

### 方案 C：按功能付费
- 基础修音 ¥99
- 自动调号 ¥49
- 高级预设包 ¥29/包
- 缺点：复杂，用户困惑

### 方案 D：企业版/公会版
- 按设备数收费
- ¥999/年/10 台设备
- 适合 MCN 和直播公会

## 定价建议

| 版本 | 价格 | 策略 |
|------|------|------|
| 免费版 | ¥0 | 引流，让用户试用 |
| 月付 | ¥29 | 低门槛，容易决策 |
| 年付 | ¥199 | 省 43%，鼓励长期 |
| 终身 | ¥499 | 限量发售，制造稀缺 |

**关键原则**:
- 免费版要能用，但不够用（让人体验到效果）
- 付费版价格要让主播觉得"一场直播就赚回来了"
- ¥29/月 对大多数主播来说不算什么
